## Overview

This repository contains the default community health files for the ZenYukti organization. These files apply to all repositories within the organization that don't have their own specific versions.

## Files and Their Purpose

| File | Purpose |
|------|---------|
| [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md) | Guidelines for participation in the ZenYukti community |
| [CONTRIBUTING.md](./CONTRIBUTING.md) | Instructions for contributing to ZenYukti projects |
| [ISSUE_TEMPLATE/*](./ISSUE_TEMPLATE) | Templates for filing bugs, feature requests, and questions |
| [PULL_REQUEST_TEMPLATE.md](./PULL_REQUEST_TEMPLATE.md) | Template for submitting pull requests |
| [SECURITY.md](./SECURITY.md) | Instructions for reporting security vulnerabilities |
| [FUNDING.yml](./FUNDING.yml) | Information about supporting ZenYukti financially |
| [workflows/*](./workflows) | GitHub Actions workflows for CI/CD |
| [copilot-instructions.md](./copilot-instructions.md) | Custom GitHub Copilot instructions |

## GitHub Actions Workflows

This repository provides standard GitHub Actions workflows that can be used across ZenYukti projects:

- **CI Workflow**: Runs tests, linting, and type-checking for JavaScript/TypeScript projects
- **Deployment Workflow**: Automatically deploys to GitHub Pages on main branch updates
- **Dependency Updates**: Regularly checks for dependency updates

## Contributing to these files

The files in this repository are meant to provide a consistent experience across all ZenYukti repositories. If you have suggestions for improvements, please follow our [contribution guidelines](./CONTRIBUTING.md).

## Local Development

To test these workflows and templates locally:

1. Clone this repository
2. Make your changes
3. Use tools like [act](https://github.com/nektos/act) to test GitHub Actions workflows locally
